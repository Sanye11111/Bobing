# 🎲 闽南博饼游戏 (Java 控制台版)

> 这是一个基于 Java 开发的控制台版博饼游戏，完整还原闽南传统博饼规则，支持自动/手动模式切换。

## 📋 项目介绍
本项目实现了 6-10 人博饼核心玩法：玩家轮流摇 6 颗骰子，程序自动判定奖项、发放奖品，并实现「状元抢夺」机制（高权值状元可替换低权值）。
- 自动模式（输入1）：无需手动操作，一键跑完所有流程并输出结果
- 手动模式（输入0）：按回车触发每轮摇骰子，还原线下博饼体验

## 📜 游戏规则
### 1. 奖项判定（6颗骰子）
| 奖项       | 骰子组合                | 权值/数量  |
|------------|-------------------------|------------|
| 六勃红     | 6个4                    | 权值7（最大）|
| 遍地锦     | 6个1                    | 权值6      |
| 六勃黑     | 6个相同非1/4点数        | 权值5      |
| 状元插金花 | 4个4 + 2个1             | 权值4      |
| 五王       | 5个4                    | 权值3      |
| 五子       | 5个相同非4点数          | 权值2      |
| 四红       | 4个4                    | 权值1      |
| 对堂       | 1-6点各1个              | 数量2      |
| 三红       | 3个4                    | 数量4      |
| 四进       | 4个相同非4点数          | 数量8      |
| 二举       | 2个4                    | 数量16     |
| 一秀       | 1个4                    | 数量32     |

### 2. 状元抢夺规则
- 首个摇出状元的玩家为「当前状元」
- 新状元权值 > 当前状元 → 抢夺成功（替换身份）
- 权值相等/更低 → 抢夺失败
- 游戏结束时持有状元身份的玩家为最终赢家

### 3. 结束条件
所有奖项库存（状元×1、对堂×2、三红×4、四进×8、二举×16、一秀×32）发放完毕。

## 🚀 快速开始
### 环境要求
- JDK 8+
- 任意控制台（IDEA Terminal/CMD/PowerShell）

### 运行步骤
1. 编译代码：`javac Bobing.java`
2. 运行程序：`java Bobing`
3. 选择模式：输入1（自动）/0（手动）
4. 输入玩家数（6-10）和姓名，等待游戏结束
5. 查看最终结果（剩余奖品、玩家获奖、最终状元）

## 🏗️ 代码结构
| 类/方法          | 功能描述                     |
|-------------------|------------------------------|
| `Bobing`          | 主类，包含 `main` 入口       |
| `static class Player` | 玩家类（姓名/获奖/状元状态） |
| `static class Prize`  | 奖品类（库存/发放/结束判断） |
| `generateSixRandom()` | 生成6颗骰子随机点数         |
| `JudgePrize()`        | 判定骰子对应的奖项           |
| `simulateEnter()`     | 自动/手动模式回车控制        |
